# 眼镜酒馆仓库约定

- 修改前优先阅读 `docs/CONTEXT_HANDOFF.md`，再按需阅读 `docs/PROJECT_STATUS.md`、`docs/ROADMAP.md` 和 `docs/CHANGELOG.md`。
- 主动管理上下文：长任务开始前、关键决策后、里程碑完成后、代理交接前或预计上下文压缩前，更新 `docs/CONTEXT_HANDOFF.md`。
- 压缩时标注优先级：P0 为当前用户决策、设计边界和安全约束；P1 为当前实现、接口、验证结果与阻塞；P2 为资产/技术参考；P3 为历史记录。不得用低优先级细节挤掉 P0/P1。
- 设计优先级：当前批准的设计文档、当前实现、近期决策、历史实验。
- 不自行发明正式配方、平衡值、顾客故事或最终美术方案；开发占位必须显式标记。
- 保持 Godot 场景模块化，以 signal 解耦系统，分离玩法状态与视觉表现。
- 不直接修改导入的 GLB；为每件资产保留稳定 ID 和玩法包装场景。
- 视觉改动必须运行项目并检查实际截图；不得只依据参数宣称完成。
- 区分“已完成”“进行中”“计划中”。没有运行验证时必须说明。
- 未经用户批准，不提交、合并、推送、发布或重写 Git 历史。
- 涉及眼镜世界、调酒操作、液体、配方、资产导入或里程碑汇报时，使用项目 Skill `develop-glasses-bar-godot`。
