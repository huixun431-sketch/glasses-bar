# 3D 资产交付与验收

## 交付格式

- 主交付为 GLB；建模源文件由美术团队自行归档。
- 1 Godot 单位等于 1 米，应用缩放与旋转，Godot 前向为 `-Z`、上方为 `+Y`。
- 使用 PBR 材质，至少提供 Base Color；需要时提供 Normal 与 ORM 纹理。
- 文件名和 `asset_manifest.json` 中的稳定 ID 必须一致。

## 功能锚点

交互资产按功能提供以下空节点，非适用锚点可以省略，但必须在清单中解释：

- `Grip`：手持位置与朝向。
- `Placement`：台面放置基准。
- `Spout`：液体流出点。
- `FillOrigin`：液面与容积视觉基准。
- `Interaction`：默认交互焦点。

## 双世界表现

默认复用同一几何模型：现实层使用资产原材质，眼镜层使用程序材质覆盖。只有轮廓确实变化时才提交独立眼镜版 GLB。客人和原材料不进入眼镜表现层。

## 接入规则

- 导入 GLB 只作为视觉子场景，不直接挂玩法脚本。
- 外层包装场景持有碰撞体、稳定 ID、锚点修正、交互脚本和材质覆盖。
- 替换占位模型只能修改资产清单路径，不能要求改写配方或交互逻辑。
- 运行 `tools/validate_assets.py assets/asset_manifest.json --allow-placeholders` 检查清单；收到 GLB 后去掉相应占位状态再验收。
