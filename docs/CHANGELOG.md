## 2026-07-21

- 用实体工具运行时状态替换“拿过即永久拥有”的布尔状态：左手至多一种放置类工具，右手至多一种手持类工具；拿起后原位消失，放下后同一实体移动到新坐标。
- 新增前后吧台连续摆放表面，使用工具占地半径阻止重叠；普通吧台在双手都有物品时优先放回右手，砧板保持左手放置类优先。
- 新增数据目录 `prototype_gameplay_catalog.tres`、`ToolCategory` 与 `OperationComplexity` 自动解析；当前工具自动分为高球杯/研钵/滤具放置类和研杵/原料勺/冰夹/水壶手持类。
- 把三个固定工序站合并为一个组合砧板；砧板按放置类工具 ID 集合解析能力，支持最多三个工具、复数工具需求和冲突组。
- 原材料改为必须经右手工具搬运；一种手持工具同时只能携带一种原材料，砧板必须先有放置类工具才能入料。
- 加冰、加水自动分类为简易工序并可在双手之间按 `R` 完成；过滤自动分类为普通工序，研磨与萃取自动分类为复杂工序。
- 将“正确答案前置门控”改为结果结算：错误手持工具与错误材料允许尝试但会生成废品；动作不足可重试；材料比例偏离决定成功概率并沿产物链传递成品完成度。
- 废品不再自动清空；玩家必须拿起对应容器或保留载料工具，与弃物桶交互后手动清理，工具本身不会被丢弃。
- 接单后直接进入制作阶段；戴镜保留为可选的信息查询/辨认步骤，不再是开始制作的前置条件。
- 新增左右手 HUD、按工具 ID 切换的手中灰盒轮廓、失败反馈视觉捕获；修复首次截图发现的手部标签遮挡和眼镜说明与手部 HUD 重叠。
- 资产占位清单从 8 项扩展为 13 项并拆分研钵/研杵，新增原料勺、冰夹、水壶和弃物桶稳定 ID；验证为 0 错误。
- 领域测试扩展至 7/7；Godot 流程测试覆盖实体拿放、防重叠、分类、砧板冲突、单一载料、错误工具、错误材料、比例完成度、手动清废、三类工序、无戴镜制作、提交和跨天重置。
