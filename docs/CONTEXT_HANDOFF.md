## P0｜不可丢失的决策与边界

- 游戏核心是“观察/记忆 → 摘镜手工操作 → 成品评价 → 经营进展”；眼镜不能自动完成制作或成为永久 HUD。
- 首个里程碑是 Windows PC、第一人称键鼠、C#、参数化液体的单杯冰美式灰盒教学关。
- 眼镜世界是独立表现场景：隐藏客人与原材料，保留吧台/工具但更换材质和信息标签；两个世界共享唯一玩法状态。
- 戴上眼镜后人物仍可自由移动；只禁止顾客、原材料和制作交互。不得再次把戴镜状态实现为移动锁。
- 正式冰美式数量、容差、奖励和经济尚未批准。开发配方必须保持 `IsPrototype=true`，不得自行发明正式内容。
- 失误采用“丢弃重做”，记录浪费与溢出；不得因模拟误差造成不可恢复的卡关。
- 未经用户批准，不提交、合并、推送、发布或重写 Git 历史。
- 主动提前压缩上下文，并按 P0/P1/P2/P3 标注重要程度；压缩不得改变完成状态或设计优先级。
- 2026-07-19 用户实际游玩反馈：玩家必须出生在吧台内；靠近可交互对象时必须出现明确提示，执行交互时必须有可感知 UI 反馈；酒馆布局可参考本轮两张预览图，但仍属于可迭代灰盒，不得据此固化最终美术或正式酒单。
- 摘下眼镜后的现实世界需启用近视模糊，初始值定义为 50 度，并允许通过内置控制台在运行时调节；戴镜世界保持清晰。
- 用户已明确授权本轮完成后创建 Git 提交；未授权推送、合并、发布或历史重写。
- 建模工具基线固定为 Blender 4.5.5 LTS，安装于 `D:\Applications\Blender 4.5 LTS\`；暂不随 winget 升级到 5.2.0，也不预装未验证的第三方插件。
- 2026-07-20 最新布局决策：正面吧台改为左侧水槽、中心砧板核心操作台、砧板旁常用工具、右侧操作手册、左右边缘展示酒；原材料统一移到后吧台，侧面设置弃物桶。操作手册只在戴镜世界显现。
- 成品不再交到独立“交付区”，而是在满足交互距离时直接与客人交互提交；不得让远距离宽容探测绕过交付距离限制。
- 磨豆机等现代流水化设备退出当前核心流程，改为传统研磨、萃取、过滤，并明确拆分“拿取工具”和“使用工具”两个环节。正式数值仍未批准，流程内容保持原型标记。
