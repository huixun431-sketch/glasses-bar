# 《眼镜酒馆》垂直切片技术设计

## 目标

首版实现单杯冰美式教学关：接单、戴镜观察、摘镜制作、交付评价和日结。正式配方尚未批准，因此开发资源只验证必要原料与步骤，所有数量均记录但不作为正式内容。

## 世界结构

- `NeutralGameplay` 保存唯一的碰撞、交互、订单、容器和液体状态。
- `RealityWorld` 显示客人、原材料和真实材质，并应用近视后处理。
- `GlassesWorld` 隐藏客人和原材料，保留吧台与工具，使用异世界材质和信息标记。
- 两个表现层使用相同实体 ID 与坐标；切换只改变表现，不复制玩法对象。
- 戴镜时允许移动、观察和读取信息，但禁止与顾客、原材料或制作台交互，不能拿取或制作。

## 教学流程

1. 在现实世界与客人交互，接受冰美式订单。
2. 按 `G` 戴上眼镜，阅读开发占位配方与工具标签。
3. 再按 `G` 摘下眼镜，按 `E` 拿取高球杯并加入冰。
4. 在水台开始倒水操作；按住鼠标左键并向下移动，或按空格/上方向键提供辅助输入；松开完成。
5. 在磨豆机执行持续研磨，再在咖啡机执行持续萃取。
6. 在出杯区交付；可随时在水槽丢弃并重做。
7. 查看评价指标和单日结算。

## 状态与数据

- `GameSession` 管理 `OrderReceived → RecipeObservation → Preparation → Delivery → Evaluation → DaySummary`。
- `DrinkWorkstation` 保存杯具、冰、液体成分、研磨、萃取、浪费和步骤记录。
- `LiquidMath` 是无 Godot 依赖的纯 C# 逻辑，保证容量上限、非负体积和溢出统计。
- `RecipeDefinition`、`IngredientDefinition`、`ToolDefinition` 与 `AssetManifest` 作为 Godot Resource 承载可配置内容。
- `RecipeEvaluator` 在原型配方中只门控必要原料和步骤；正式目标量与容差存在时才计算数量通过率。

## 边界

本里程碑不包含莫吉托、随机订单、多顾客 AI、升级购买、正式经济、长期存档、800 度渐进难度或真实流体物理。
