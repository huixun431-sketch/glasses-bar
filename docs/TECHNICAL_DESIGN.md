# 《眼镜酒馆》垂直切片技术设计

## 目标

首版实现单杯冰美式教学关：接单、戴镜观察、摘镜制作、交付评价和日结。正式配方尚未批准，因此开发资源只验证必要原料与步骤，所有数量均记录但不作为正式内容。

## 世界结构

- `NeutralGameplay` 保存唯一的碰撞、交互、订单、容器和液体状态。
- `RealityWorld` 显示客人、原材料和真实材质，并应用近视后处理。
- `GlassesWorld` 隐藏客人和原材料，保留吧台与工具，使用异世界材质和信息标记。
- 两个表现层使用相同实体 ID 与坐标；切换只改变表现，不复制玩法对象。
- 戴镜时允许移动、观察和读取信息，但禁止与顾客、原材料或制作台交互，不能拿取或制作。

## 教学流程

1. 在开场菜单选择“开始营业”，进入第 1 天；菜单关闭前玩法输入保持禁用。
2. 在现实世界与客人交互，接受冰美式订单。
3. 按 `G` 戴上眼镜，阅读仅在眼镜世界显现的操作手册和工具标签。
4. 摘镜后从后吧台拿取杯具、冰和咖啡豆；从前台拿取研钵与研杵。
5. 在中央砧板持续研磨；拿取传统滤具后，依次完成浸润萃取和过滤入杯。
6. 在左侧水槽手工加水并处理冰；任意制作阶段都可在侧面弃物桶丢弃重做。
7. 拿着成品走近客人并交互提交；距离超过当前原型上限时拒绝交付，不再使用独立出杯区。
8. 查看评价和日结；按 Enter 开始下一天并重置当日玩法状态。

## 状态与数据

- `GameSession` 管理菜单开始门控、当前天数与 `OrderReceived → RecipeObservation → Preparation → Delivery → Evaluation → DaySummary`。
- `DrinkWorkstation` 保存杯具、冰、液体成分、研钵/滤具拿取、咖啡豆、研磨、萃取、过滤、浪费和步骤记录。
- `LiquidMath` 是无 Godot 依赖的纯 C# 逻辑，保证容量上限、非负体积和溢出统计。
- `RecipeDefinition`、`IngredientDefinition`、`ToolDefinition` 与 `AssetManifest` 作为 Godot Resource 承载可配置内容。
- `RecipeEvaluator` 在原型配方中只门控必要原料和步骤；正式目标量与容差存在时才计算数量通过率。

## 边界

本里程碑不包含莫吉托、随机订单、多顾客 AI、升级购买、正式经济、长期存档、逐日难度/奖励变化、800 度渐进难度或真实流体物理。
