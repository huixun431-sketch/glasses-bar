## 一、已完成的事项

### 工程与架构

- 已从空目录建立 Godot 4.7.1-stable .NET/C# 项目，使用 Forward+，目标平台为 Windows PC。
- 已建立模块化目录：场景、脚本、领域逻辑、数据资源、Shader、资产、工具、测试和文档。
- 已实现唯一权威玩法状态：订单、配方步骤、杯具、液体、浪费、评价等数据不归属于任一视觉世界。
- 已实现 `GameSession` 状态机：接单 → 配方观察 → 制作 → 交付 → 评价 → 日结。
- 已建立信号通信和接口层：`IInteractable`、`IManualOperation`、`IWorldPresenter`、`ILiquidContainer`。

### 核心玩法

- 已实现第一人称键鼠控制、视线射线、交互提示、拿杯和手持杯表现。
- 已根据 2026-07-19 实际游玩反馈把玩家出生点移到吧台内侧工作区，初始朝向客人与正面工作台；不再从吧台外开始。
- 交互检测已升级为中心射线加宽容 ShapeCast：靠近并大致朝向交互处即可显示提示，不需要像素级对准。
- HUD 已增加明确的交互状态：可用/不可用原因、准星状态变化、连续操作进度条、接单/拿杯/加冰/完成操作后的短时确认条。
- 已实现现实世界与眼镜世界两个独立表现场景：
  - 现实世界显示客人和原材料，并应用近视模糊。
  - 眼镜世界隐藏客人与原材料，保留吧台和工具，使用异世界材质及信息标签。
  - 两个世界共享同一物体位置与玩法进度，切换不会复制物理状态。
- 戴眼镜时仍允许人物移动和观察，但禁止与顾客、原材料及制作台交互，不能拿取或制作；摘镜后恢复现实交互。

- 已实现连续手工操作：鼠标动作与键盘辅助共同控制倒水、研磨和萃取，不是单次按钮自动完成。
- 已实现参数化液体：容量、成分、流速、倾角输入、溢出与浪费记录。
- 已实现丢弃重做机制；错误不提供撤销，但不会因模拟误差造成不可恢复卡关。
- 已实现原型评价：正式用量未批准前，只判定必要原料和步骤；数量数据会记录但不作为正式评分。

## 二、关键决策

### P0｜玩法与产品方向

- 核心循环固定为“观察/记忆 → 摘镜手工操作 → 成品评价 → 经营进展”。
- 首个里程碑是单杯冰美式灰盒垂直切片，不扩展莫吉托、随机经营日或完整经营系统。
- 目标体验为 Windows PC、第一人称键鼠、C#、参数化拟真液体。
- 眼镜中的画面是平行世界，不是普通 AR 叠加或暂停菜单。
- 眼镜世界隐藏客人和原材料；吧台、工具保留但材质不同，并提供配方和标签信息。
- 眼镜只帮助观察、记忆和规划，不能自动完成任务、代替玩家技能或成为永久 HUD。
- 戴镜不锁定人物移动；玩家可在眼镜世界走动观察，但顾客、原材料和制作交互全部禁用。
- 液体优先级为玩家体验、反馈、稳定性、物理真实性；不采用真实流体作为评分依据。
- 操作采用混合手工方案：普通交互用按键，倒液体/研磨/萃取使用连续鼠标输入并提供键盘辅助。
- 制作失误允许丢弃重做，记录材料和时间损失，不允许模拟错误形成死局。
- 正式配方、容差、奖励、顾客故事和最终美术不能由程序自行发明。
- 用户提供的两张预览图用于当前灰盒空间布局和交互 UI 方向参考，不冻结最终陈设、美术或酒单内容。
- “近视 50 度”在当前版本作为运行时玩法参数保存；Shader 使用稳定的屏幕空间模糊与轻微边缘色差近似，不宣称医学或真实光学模拟精度。
- 模糊参数属于现实世界表现状态，不进入配方/液体等权威玩法状态；戴镜仅切换表现层，不能自动操作。

### P0｜开发与授权边界

- 正式冰美式数据交付前，继续使用显式原型资源，不能将占位量视为批准配方。
- 导入 GLB 必须包装在手写玩法场景中，不直接修改或给导入节点绑定核心玩法。
- 不因上下文压缩把“已实现但未验证”提升为“已完成”。
- 未经用户批准，不提交、合并、推送、发布或重写 Git 历史。
- 用户已在 2026-07-19 明确授权初始化仓库并提交当前成果；该授权不自动延伸到后续提交、推送、合并、发布或历史重写。

### P1｜技术实现决策

- Godot 基线为 4.7.1-stable .NET，目标框架为 .NET 8。
- 配方、原料、工具、容差和资产路径全部数据驱动；正式内容不能硬编码。
- 现实/眼镜表现与中立玩法状态分层；切镜只切换表现层。
- 纯领域逻辑放在 `src/Domain/`，保持无 Godot 依赖，便于 NUnit 测试。
- 测试必须检查明确 PASS 日志，不能只依据 Godot 进程退出码判断成功。
- 导出同样必须检查日志中的 `ERROR`/异常；Godot 4.7.1 在部分 .NET 导出错误下仍可能返回退出码 0。
- Debug/Release 必须导出到独立目录，避免共享的 .NET 数据目录被后一次导出覆盖。
- 美术替换默认复用几何并覆盖眼镜材质；只有轮廓变化时才需要独立眼镜版 GLB。
