## 二、关键决策

### P0｜玩法与布局

- 核心循环保持“观察/记忆 → 摘镜手工操作 → 成品评价 → 天数推进”。
- 左右位置始终以玩家站在吧台内、面向客人的第一人称画面为准：水槽在画面左侧，眼镜操作手册在画面右侧。
- 砧板是中央核心操作台；前台不再排列大型现代设备，也不设置独立交付区。
- 操作手册属于眼镜世界信息表现，不保存权威配方状态，也不能自动完成操作。
- 原材料位于后吧台并在眼镜世界隐藏；左右展示酒目前只是陈设占位，不代表正式酒类选择。
- 成品必须走近客人提交。宽容 ShapeCast 只改善瞄准和提示，不得绕过交付距离。
- 传统工序必须区分拿工具与使用工具，不能把“点击设备”替换成另一种一键自动制作。

### P0｜内容边界

- 冰美式仍是开发占位配方，当前步骤、用量、容差、评分、奖励和经济均未获正式批准。
- 天数系统目前只负责循环与重置，不自行引入逐日难度、顾客故事、收入、成本或视力增长。
- 当前所有器具、酒瓶、顾客、建筑和材质仍是灰盒，不代表最终美术方案。
- 失误继续使用“丢弃重做”，记录浪费但不形成不可恢复死局。

### P1｜技术实现

- `GameSession` 管理游戏是否开始、当前天数、世界模式和日流程；工作台保存唯一饮品/工具状态。
- `OpeningMenuController` 只负责菜单表现和请求信号；主场景协调玩家、工作台与会话重置。
- 眼镜切换只切表现层，不复制工具、液体、配方或天数状态。
- 视觉修改必须用实际 Forward+ 截图检查；测试与导出日志必须出现明确 PASS/无错误证据。
- 本轮用户授权了每个分支的独立 Git 提交；未授权推送、合并、发布或历史重写。

## 三、未完成的待办

### P1｜下一轮人工验收

1. 使用新版 Release 包完整游玩至少两天，确认开场菜单、接单、观察手册、后吧取料、拿工具、研磨、萃取、过滤、加冰/加水、走近客人提交、日结和下一天。
2. 重点记录：左右动线是否自然、后吧取料是否过远、3.15 米提交距离是否合适、中央三个传统操作是否仍显拥挤、工具拿取反馈是否足够、50 度模糊是否舒适。
3. 根据实玩反馈继续调整交互距离、碰撞、标签位置、操作节奏和鼠标手感。

### P1｜外部内容阻塞

- 等待策划批准正式冰美式步骤、用量、容差、评价权重、奖励和成本。
- 等待正式酒类清单、工具使用说明与操作手册内容；当前手册只显示开发占位流程。
- 等待首批 GLB 与纹理；需要优先制作/替换吧台、水槽、砧板、研钵与研杵、传统滤具、杯具、冰桶和咖啡豆。

### P2｜后续功能

- 开场菜单尚无设置、存档、继续游戏、音量和键位页面。
- 天数尚未接入正式经营数据、随机订单、多顾客、难度变化或存档。
- 传统研磨/萃取/过滤目前使用稳定参数化操作，不是最终动画、声音或物理表现。
- 莫吉托、搅拌/捣压、长期经营和视力进展仍属于后续里程碑。
